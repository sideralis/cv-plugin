Known Issues

Refreshing workspace never finishes

Kill Eclipse and restart.

It can also be linked to the fact that clearcase plugin connection is already started. Go to Clearcase menu and press Connect to Rational Clearcase, in order to have all other clearcase menus grayed.

Eclipse does not work properly anymore

Delete the workspace and restart.

Projects do not reopen at startup

It seems that with Windows7 and when project are located on a clearcase drive, when Eclipse is started, previously open projects do not reopen due to a missing .project file whereas it is existing. In this case, you need to delete the project (w/o deleting the file), then import\existing project into workspace.

You try to open a text file (*.scf for ex) with Eclipse but it does not open

Go to Windows\Preference\General\Content Types

Click on Text.

Click on Add...

Type *.scf

Then press OK up to the end.

Now you should be able to open a scf file as a text file.

In Problem view, the error “Reading config file” is always here even if there is no error during compilation

This is due to the fact that a bad error parser is activated by default with Eclipse. You must deactivate it.

Right click on the project and select Properties

Go to C/C++ Build -> Settings.

In the Error Parsers tab, unchek all parsers except ARM Error Parser.

Press OK.

 

All __asm {instruction} lines are reported as syntax error in Eclipse

Goto  Windows -> Preferences -> C/C++ -> Language Mappings and check that

 Content type: C source file is mapped to Language: ARM extensions, if not you need to add it.

The following syntaxes are supported by Eclipse and by ARM compiler.

Multiples lines:

__asm return-type function-name(parameter-list)

{

    // ARM/Thumb/Thumb-2 assembler code

    instruction[;instruction]

    ...

    instruction

}

Or single lines:

__asm (“instruction[;instruction]”);

__asm{instruction[;instruction]}
