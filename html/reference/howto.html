Reference

Refactoring

This means that you can rename a variable or a function and that this rename will apply to all usages in code of this variable/function.

To do it, select the variable/function you want to refactor, right click on it, and select Refactor.

Find usage/reference of a function/variable

Right click on the function/variable then select References -> Project

Go to function declaration

Highlight the function and press F3

Open include file

Select the name of the file in the #include line, right click and click Open Declaration.

Automatically add include

Right click on a function, select Source, select Add Include.

Complete words

Press the beginning of a variable then Ctrl+space

Comment lines

Select the lines you want to comment, then press Ctrl+/. Do the same for uncommenting.

Tile windows vertically

Click on the name bar of the window. Let the mouse button pressed and dragged the bar to the left until you see a shadow window on the right which should be half the size of the current window size. Release button. You should have now 2 editor windows tiled vertically.

Duplicate a window

Right click on the title bar of the source window and select New Editor

Add tasks

Just add a comment with a TODO or FIXME tag like

// TODO: add toto

// FIXME correct this code

then if the task window is not already shown, you can display it by going to menu Window\Show View\Task

Format source code

Press Ctrl+Shift+F

How to avoid line wrapping

Go in menu Window -> Preferences -> C/C++ -> Code Style.

Create a new profile based on K&R and edit it.

Go to Line Wrapping tab.

Change Maximum line width to 200 (for example). This means that all lines containg more than 200 characters will be wrapped on 2 or more lines. You can put more if you really don't want any wrapping at all.

Change Line wrapping policy to Do not wrap for all items as Class declarations, Function declaration, 'enum' declaration, function calls and Expressions.

Find text

Go in the Search menu, you can then search either C code (C\C++...') or any text (Search...)

If you want to search in a specific folder which is not included in your project, then do the following:

Right click on your project

Select new Folder.

Click advanced and check Link to folder in the file system.

Then browse to your folder and click ok.

Now each search in your project, will also search in the added folder(s)

Find the matching brace

Double click on the character after the brace. Then all code between starting and closing brace will be highlighted.

Hide the code which is not defined

All code between a #ifdef and #endif can be shown with a gray background if the variable is not defined. To do that, go to Properties of project. Go to C\C++ general then to Paths and Symbols. Then to Symbols tab. You then need to add the variable you want to define and put the value you want.

It is not possible to make a link to what you have defined in the makefile.

Compare files

Click on the files you want to compare (press Ctrl+click to have both files selected). Then right click on a file and select Compare -> Each Other.

Save sources before build

Go to Windows -> Preferences -> General -> Workspace and check Save automatically before build

Correction of spelling errors

By default Eclipse will correct any English spelling error. The error will be underline in red. Put the mouse cursor on it and wait. A new window will appear after few seconds and will offer you some choices.

Add new include files and source paths

It may happen that pressing on F3 on a function or a variable does not bring you to the declaration of the object. The cause is that Eclipse does not know where to look for finding the declaration because the function or variable is declared in a non trivial directory (not in _base, _lib or testcase folder usually).

You can fix this by adding some more directories to look for.

For missing includes

Right click on the project and select Properties

Go to C/C++ General -> Paths and Symbols

In the include tab, add the missing include folders

For missing sources

Right click on the project and select New -> Other...

Select C/C++ -> Folder

Click on advanced and check Link to folder in the file system

Browse to the missing source folder.

Repeat these steps if you need to include more than one folder

Edit in column mode

Press Alt+Shift+A to toggle column mode. In this mode, you can select text as a rectangle, and not anymore as consecutive lines.

This means that for example, in this text:

#include "toto/file1.h"

#include "toto/file2.h"

You can select both toto and delete them. It will give you then this

#include "/file1.h"

#include "/file2.h"

Or you can even select a thin square between the end of toto and before the /, press 1 and then you will get this

#include "toto1/file1.h"

#include "toto1/file2.h"
